# Going_Scan 项目综述

## 一句话定义

Going_Scan 是一个 **Go 语言编写的高性能网络资产扫描与分析平台**，由 GS（高速发现引擎）、gping（定向验证探针）和 UAM（统一资产模型）三个子系统协同构成，形成"发现、沉淀、验证、扩展"的完整资产侦察工作流。

---

## 项目要解决什么问题

传统扫描器通常在两个方向上做取舍：要么偏重吞吐、把结果压缩成"端口开没开"；要么偏重识别、把大量时间消耗在逐连接探测上。

Going_Scan 把这两件事拆开并串联：

- **L4 层**：高速、无状态、可控的原始探测，只管快速发现
- **L7 层**：只消费可信结果，做服务识别与指纹提取
- **输出层**：统一聚合事实流，生成结构化画像
- **资产层 (UAM)**：沉淀所有发现为可追溯的资产知识
- **验证层 (gping)**：对可疑端点做定向确认，回写资产状态

最终目标不是"做一个更强的扫描器"，而是**形成一个可追溯、可确认、可扩展的资产系统**。

## 三工具协同架构

### 协作关系

三个工具构成一条清晰的资产处理流水线：

1. **GS 高速发现**：大规模 L4 端口扫描 + L7 服务识别，产出主机、端口、服务等初步资产事实，写入 UAM 的 Observation 与 Claim
2. **UAM 资产中枢**：接收 GS 的发现结果，统一管理资产身份（Host/Endpoint）、存储原始观察（Observation）、归一断言（Claim）、维护当前最佳视图（Projection），并提供查询接口
3. **gping 定向验证**；从 UAM 读取资产上下文，对目标执行精确认证（支持 Raw/Stack/App 三路线），验证结论回写 UAM 的 Claim 和 Projection

三个工具共享同一个 SQLite 数据库（uam.db）作为数据交换中心。

### 分工

| 工具 | 角色 | 核心使命 |
|------|------|----------|
| **GS** | 高速发现前端 | 大规模、高吞吐的自动资产发现 |
| **UAM** | 状态中枢 | 统一身份、观察、断言、视图、扩展 |
| **gping** | 定向验证器 | 人工驱动的精准确认、修订、覆盖 |

GS 负责"广撒网"，gping 负责"精确认"，UAM 负责"长期记"。三者职责明确，互不重叠。

## 核心技术特色

### 1. PacketTensor 极限压缩分类

收到的回包不会被逐层深度解析，而是被快速压缩为一个 **32 位的轻量状态张量**（`uint32`），用 O(1) 位运算完成 TCP/UDP/ICMP 端口状态判定。全程零 GC 压力，是 L4 引擎的性能基石。

位布局：

- bits 0-7：TCP Flags / ICMP Type
- bits 8-15：IP TTL
- bits 16-19：Window Size（量化值）/ ICMP Code
- bits 20-27：IP Protocol
- 特殊值 0xFFFFFFFF：超时魔术字

### 2. CWND 自适应并发控制

模仿 TCP 拥塞控制算法：成功收到回包时增加并发窗口，超时时乘性减半。引擎的并发度随网络状况自动调节，既保证吞吐又避免拥塞。配合 SRTT 指数平滑和 RTO 动态计算，形成完整的自适应控制闭环。

### 3. Channel ID 物理匹配

将通道 ID 编码到发包源端口号中，收包时直接从回包的目的端口 O(1) 定位到对应探针，无需查表或遍历。配合 64 位校验码（srcIP + srcPort）原子比对，确保回包精确匹配到发起探针。

### 4. 零分配发包流水线

通过 `sync.Pool` 复用 128 字节发包缓冲区，直接在池化内存上构建 L2/L3/L4 报文，配合 Lock-Free Ring Buffer（65536 容量）完成高吞吐结果流的无锁传递。

### 5. 五层 UAM 资产模型

UAM 采用严格的分层设计，每层职责不可混淆：

- **第一层 Identity**：Host（主机，以 IP 为身份）和 Endpoint（端点，以 IP + Protocol + Port 为身份）
- **第二层 Observation**：原始观察，记录"哪个工具、在什么时间、看到了什么"
- **第三层 Claim**：归一断言，记录"这个观察意味着什么资产结论"（端口状态、服务识别、可达性）
- **第四层 Projection**：当前投影，由 Claim 推导的当前最佳视图（HostProjectionCurrent / EndpointProjectionCurrent）
- **第五层 Extension**：扩展挂接，专项扫描的深结果进入 ModuleResult，不污染核心资产模型

关键设计原则：
- **Observation 不等于 Claim**：观察记录"发生了什么"，断言记录"这意味着什么"
- **Projection 不等于历史**：当前视图由 Claim 推导，不是最后一次 Observation 的直接覆盖
- **Claim 优先级体系**：override(4) > manual(3) > observed(2) > inferred(1)，确保人工确认不被自动扫描覆盖

### 6. gping 三路线统一验证

gping 不是 GS 的附属品，而是一个独立的多路线探针引擎。三条路线内部实现完全不同，但共享统一生命周期：Resolve（解析目标）、Plan（规划动作）、Prepare（准备执行）、Execute（执行探测）、Interpret（解释证据）、Emit（输出结果）。

| 路线 | 定位 | 典型方法 |
|------|------|----------|
| **Raw** | 绕过 OS 协议栈的包级控制 | tcp-syn, icmp-echo-raw |
| **Stack** | 使用 OS 协议栈的真实通信 | tcp-connect, tls-handshake, banner-read |
| **App** | 应用层协议适配器 | HTTP/DNS/SSH/MySQL/Redis/FTP/SMTP |

### 7. 模板化验证工作流

gping 内置 16 个 YAML 模板，支持 `${var}` 变量展开、条件执行（`when` DSL）、多动作编排、提取规则（`extracts`）和推荐规则（`recommend`），将常见验证流程标准化。用户可以一条命令完成"TCP 连接 + TLS 握手 + HTTP 请求 + 结果写入 UAM"的完整确认流程。

## 支持的扫描能力

### L4 扫描模式
- **TCP SYN**：半开扫描（默认），最常用的端口扫描方式
- **TCP ACK**：防火墙规则探测，判断端口是否被过滤
- **TCP Window**：利用 TCP Window 字段辅助判定端口状态
- **UDP**：UDP 端口状态探测
- **主机探活**：ICMP Echo / TCP / UDP 多协议探活联动，探活结果实时推进端口扫描

### L7 服务识别
- 内嵌 nmap-service-probes 探针数据库
- 正则预编译 + 端口索引加速匹配
- 提取字段：service, product, version, info, hostname, os, device, cpes
- 支持 TCP/UDP、SSL 端口、fallback 探针、rarity 分级

### gping 应用层适配器
HTTP(S)、DNS、SSH（含 ECDH 密钥交换 + SHA256 主机密钥指纹）、MySQL（含 MariaDB/Percona flavor 检测）、Redis（RESP 协议 + INFO 解析）、FTP（RFC 959 多行回复）、SMTP（含 STARTTLS 证书提取）

## 典型工作流

### 基础场景：快速扫描

```bash
./goscan scan 192.168.1.0/24 --syn -F -V -o report.json
```

对 C 类网段执行快速 SYN 扫描 + 服务识别，结果导出为 JSON 画像。

### 完整场景：GS + UAM + gping 资产闭环

**第一步**：GS 发现资产并沉淀到 UAM

```bash
./goscan scan 192.168.1.0/24 --syn -V --uam-db uam.db
```

**第二步**：查看发现的可疑 HTTPS 端点

```bash
./goscan gping candidates --uam-db uam.db --uam-service https
```

**第三步**：定向验证并确认

```bash
./goscan gping --uam-db uam.db --uam-service https --pick-index 1 \
  --template uam/https-enrich --assert confirmed
```

**第四步**：查看主机的完整资产报告

```bash
./goscan uam report --db uam.db --ip 192.168.1.10
```

## 技术栈与运行要求

- **语言**：Go 1.24+
- **包依赖**：libpcap / npcap（需原始发包与抓包权限）
- **存储**：SQLite（WAL 模式 + 外键约束）
- **权限**：Linux/macOS 需 root 或等效权限运行扫描
